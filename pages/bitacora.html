Registro del proceso

Bitácora del Proyecto

Documentamos las decisiones de diseño, dificultades y aprendizajes a lo largo del TP1.

## Planificación

Definición del proyecto y reparto de tareas

En la primera reunión definimos que íbamos a crear una agencia web ficticia llamada DevTeam. Decidimos usar una paleta oscura con acentos en violeta para diferenciarnos de los sitios genéricos. Dividimos las responsabilidades por áreas: Carlos en la portada y estructura, Maximiliano en el navbar, Juan en los perfiles y Sofía en la sección de servicios.

## Configuración

Creación del repositorio y estructura de archivos

Creamos el repositorio en GitHub y definimos la estructura de carpetas siguiendo las buenas prácticas del TP: index.html en la raíz, css/ para estilos, js/ para scripts e img/ para imágenes. Cada integrante creó su rama de trabajo (feature/portada-carlos, feature/navbar-ana, etc.).

## Desarrollo

Decisiones de diseño: tipografía y colores

Elegimos Roboto (Google Fonts) para títulos y elementos de código, y DM Sans para el cuerpo. El fondo oscuro (#0a0a0f) con acento violeta (#6c63ff) da un tono tech sin ser cliché. Definimos variables CSS para mantener consistencia en todo el proyecto.

## Dificultades

Merge conflicts al integrar ramas

Al mergear las ramas individuales al main, encontramos conflictos en el archivo CSS. La solución fue acordar una única hoja de estilos compartida (styles.css) y que cada integrante usara clases locales en su sección en vez de redefinir las globales. También aprendimos a usar git pull --rebase para mantener el historial limpio.

## Responsive

Implementación del diseño responsive

Agregamos los tres breakpoints obligatorios: 400px (móvil), 900px (tablet) y 1200px (escritorio). El mayor desafío fue hacer que el menú de navegación colapsara en un botón hamburguesa en pantallas pequeñas, lo cual requirió un pequeño script JavaScript para toggle de clases.

## Finalización

Deploy en Vercel y checklist final

Publicamos el sitio en Vercel conectando el repositorio de GitHub. Revisamos todos los requerimientos del TP: navegación entre páginas sin usar el botón atrás, funciones JS en la portada y en cada perfil, README con toda la información requerida y la bitácora completa.
